ShipTrackr Project Report

A modern, client-side shipment and task management platform for logistics teams. Built for speed, clarity, and real-world usability.

  • Fully responsive & accessible
  • Dark mode & modern UI
  • Interactive analytics & calendar
Download PDF Report

Project Specifications

  • Five pages: Home, Tasks, About, Contact, and a custom page (FAQ).
  • All DOM manipulation done with jQuery.
  • Well-formed HTML with appropriate metadata.
  • Fully responsive design using Bootstrap.
  • Sticky navigation bar with logo and links.
  • Dark mode toggle on all pages.
  • Task management: add, edit, delete, mark complete, filter, sort, and prioritize tasks.
  • Summary and analytics of tasks (total, pending, completed, by priority).
  • About page with team grid and carousel (team is fictional).
  • Contact page with form, Google Maps, and social media links.
  • Custom FAQ page with at least three components.
  • Accessibility: ARIA labels, alt text, and proper meta tags.
  • Client-side form validation for all forms.
  • Version control with Git and GitHub.
  • Ready for in-class presentation and code explanation.

How the Requirements Were Met

  • Home Page: Features a welcome section, latest activity, sticky navbar, and dark mode toggle. The call-to-action button links to the Tasks page.
  • Tasks Page: Users can add, edit, delete, and mark tasks as completed. Tasks can be filtered and sorted. Priorities are color-coded and filterable. A summary section shows total, pending, and completed tasks.
  • About Page: Includes a fictional team grid with avatars and a Bootstrap carousel for testimonials.
  • Contact Page: Contains a validated contact form, Google Maps embed, and social media icons. Form submissions appear as pop-ups.
  • FAQ Page: Accordion for common questions, a roadmap timeline, and onboarding cards.
  • All Pages: Use Bootstrap for layout and responsiveness, and jQuery for all DOM manipulation. Navigation and footer are consistent. Accessibility and ARIA standards are followed.
  • Version Control: All code is tracked with Git and available on GitHub.
  • Presentation Ready: The site is ready for live demonstration and code walkthrough in class.

Coding Decisions

  • jQuery for DOM Manipulation: All dynamic updates (adding, editing, deleting tasks, updating tables, toggling dark mode) are handled with jQuery for simplicity and assignment compliance.
  • Bootstrap for Layout: Used for grid, cards, navbar, forms, and responsive design. Ensures a modern look and mobile compatibility.
  • localStorage: Used for client-side data persistence, so tasks and shipments remain after page reloads.
  • Component Reuse: Navbar and footer are reused across all pages for consistency.
  • Accessibility: ARIA labels, alt text, and keyboard navigation are implemented where possible.
  • Form Validation: All forms use client-side validation for required fields and correct formats.
  • Bonus Features: Priority color-coding, FAQ accordion, onboarding cards, and a roadmap timeline on the FAQ page.

Lessons Learned

  • jQuery is powerful for DOM manipulation, but can get verbose for complex UIs.
  • Bootstrap makes responsive design much easier, but customizing components for a unique look takes extra effort.
  • Ensuring accessibility and good UX requires attention to detail (ARIA, alt text, keyboard navigation).
  • Testing on different devices and browsers is essential for a polished result.
  • Version control (Git) is invaluable for tracking changes and collaborating, even on solo projects.

Future Improvements

  • Add user authentication and multi-user support.
  • Implement backend integration for persistent, cross-device data.
  • Expand analytics with more advanced visualizations and export options.
  • Enhance accessibility with keyboard shortcuts and improved ARIA roles.

Reflections

This coursework was completed individually and served as a valuable opportunity to deepen my frontend development skills. Taking full ownership of the project meant managing everything from planning and interface design to coding, testing, and deployment. One of the most challenging aspects was using jQuery for all DOM manipulation, which required a different mindset than working with modern frameworks. Ensuring consistency in layout, responsiveness, and dark mode integration across multiple pages added complexity, especially when coordinating Bootstrap components and localStorage behavior.

I particularly enjoyed building out dynamic features like the interactive calendar, analytics charts, and form modals, and seeing how small UI tweaks could improve the overall user experience. The project helped reinforce core concepts such as responsive grid systems, modular JavaScript structuring, and accessibility best practices using ARIA labels and semantic HTML. Managing version control with Git also played a key role in maintaining an organized workflow. Overall, I'm proud of the final result and feel more confident moving forward into more advanced development projects involving frameworks like React or Vue.
